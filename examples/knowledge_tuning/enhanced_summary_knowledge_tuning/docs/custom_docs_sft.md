# Custom Documents SFT Results

This section demonstrates knowledge tuning on **custom, domain-specific documents** — in this case, the [BMO financial document](../document_collection/bmo.pdf). The goal is to inject document-specific knowledge into a model via SFT using synthetic data generated by the pipeline.

---

## Experiment Setup

- **Student model:** `meta-llama/Llama-3.1-8B-Instruct`
- **Teacher models (for data generation):** `Llama 3 70B`, `GPT-OSS-120B`
- **Hyperparameters (SFT runs):** learning rate 5e-6, batch size 16, 7 epochs

---

## Evaluation: RAG Setup

We evaluate the fine-tuned models in a **retrieval-augmented generation (RAG)** setting using two configurations to measure both accuracy and robustness:

- **RAG Score** — The vector store contains only the documents the model was trained on. This measures how well the model leverages its injected knowledge when relevant context is retrieved.
- **Large Scale RAG Score** — The vector store contains **all BMO documents**, not just the ones used for training. This tests the model's robustness when retrieved context includes both familiar and unseen documents, simulating a more realistic production setting.

---

## Results

| Training Method        | Teacher Model                     | Student Model                    | RAG Score | Large Scale RAG Score |
| ---------------------- | --------------------------------- | -------------------------------- | --------- | --------------------- |
| Baseline (no-training) | —                                 | meta-llama/Llama-3.1-8B-Instruct | 205.5     | 201                   |
| SFT                    | meta-llama/Llama-3.3-70B-Instruct | meta-llama/Llama-3.1-8B-Instruct | 215       | 213                   |
| SFT                    | openai/gpt-oss-120b               | meta-llama/Llama-3.1-8B-Instruct | **221.5** | 219                   |

---

## Key Takeaways

- **Baseline prompting** (no fine-tuning) scores 205.5 on RAG, showing the model has some general knowledge but lacks document-specific details.
- **SFT with Llama 3 70B as teacher** improves RAG score to 215, a clear gain over the baseline from the enhanced summary pipeline.
- **Using a stronger teacher model** (`GPT-OSS-120B`) for data generation pushes performance further to **221.5**, demonstrating that higher-quality synthetic data translates directly to better downstream results.
- **Large Scale RAG scores** remain close to RAG scores across all experiments (e.g., 221.5 → 219), showing the fine-tuned models are robust even when the retrieval pool is diluted with unseen documents.
